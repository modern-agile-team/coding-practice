# 코딩테스트 연습 (Level 0~10)

프로그래머스/백준 난이도 기준으로 설계한 TypeScript 코딩테스트 연습 문제 모음입니다.
**Level 0(언어 기본기)** 부터 시작해, 입문(프로그래머스 Lv0 / 백준 Bronze)을 거쳐
중급(프로그래머스 Lv3 / 백준 Gold)까지 난이도 곡선을 따라 배치되어 있습니다.
(대상: 코딩 입문자 ~ 취준 코딩테스트 중급)

## 학습 방식

- 각 문제 폴더에는 **`solution.ts`(빈 스텁)** 와 **`solution.test.ts`(테스트)** 가 있습니다.
- 모든 문제의 구현 대상 함수는 **`solution`** 으로 통일되어 있습니다. (프로그래머스와 동일한 방식)
- 입출력은 stdin/stdout이 아니라 **함수 인자 → 반환값** 입니다.
- `solution.ts` 안의 `solution` 함수를 직접 구현해 테스트를 통과시키면 됩니다.
- 구현 전에는 `throw new Error("아직 구현되지 않았습니다")` 때문에 테스트가 실패합니다.
- 문제 설명은 `solution.ts` 최상단 주석에 **최소한의 요구사항과 예시 1개**만 있습니다.
- 엣지 케이스 등 세부 동작은 `solution.test.ts`의 테스트가 곧 명세입니다. 막히면 테스트 이름과 케이스를 읽으세요. (실무처럼!)

## 브랜치 룰

여러 명이 같은 레포에서 각자 문제를 풀기 때문에, **기수와 이름이 들어간 브랜치**에서 작업합니다.
`main`은 문제 원본 유지용이므로 **직접 푸시하거나 main으로 PR을 올리지 마세요.**

| 구분 | 형식 | 예시 |
|------|------|------|
| 베이스 브랜치 (본인 풀이 모음) | `{기수}-{영문이름소문자}` | `14-minsoonki` |
| 작업 브랜치 (문제 하나) | `{기수}-{영문이름소문자}-{문제폴더명}` | `14-minsoonki-7.2-나무자르기` |

- **기수**는 숫자만 씁니다. (예: `14`, `15`)
- **이름**은 영문 소문자로 씁니다. (예: `minsoonki`)
- **문제 부분**은 문제 폴더명을 그대로 씁니다. (예: `0.1-자료형-타입판별`, `7.2-나무자르기`)
- 슬래시(`/`) 대신 전부 하이픈(`-`)으로 연결합니다.

### 작업 순서

```bash
# 1. 처음 한 번: main에서 본인 베이스 브랜치 생성
git switch main && git pull
git switch -c 14-minsoonki
git push -u origin 14-minsoonki

# 2. 문제 풀 때마다: 베이스 브랜치에서 작업 브랜치 생성
git switch 14-minsoonki
git switch -c 14-minsoonki-7.2-나무자르기
```

3. 풀이가 끝나면 작업 브랜치를 푸시하고, **본인 베이스 브랜치로 PR**을 올립니다.
   (예: `14-minsoonki-7.2-나무자르기` → `14-minsoonki`)
4. PR을 올리면 CI가 **변경한 문제의 테스트만** 자동으로 실행합니다. **테스트가 모두 통과해야 머지할 수 있습니다.**
5. PR이 머지되면 베이스 브랜치에 본인 풀이가 차곡차곡 쌓입니다.

## 실행 방법

### 1. 처음 한 번 (필수)

```bash
npm install
```

### 2. 풀고 싶은 문제 골라서 실행 (추천)

```bash
npm start
```

명령을 치면 메뉴가 뜹니다. **화살표 키 ↑↓로 이동하고 Enter로 선택**하세요.

1. **레벨 선택** — `Level 00 ~ Level 10` 중 하나를 고릅니다. (맨 위 `전체`를 고르면 모든 문제를 한 번에 실행)
2. **문제 선택** — 그 레벨 안의 문제를 고릅니다. (맨 위 `전체`를 고르면 그 레벨 전체를 실행)

선택한 문제의 테스트만 실행되므로, **아직 안 푼 다른 문제 때문에 실패가 잔뜩 뜨는 일이 없습니다.**
`solution.ts`의 `solution` 함수를 구현한 뒤 다시 `npm start`로 해당 문제를 골라 통과를 확인하세요.

> 취소하려면 `Ctrl+C`를 누르면 됩니다.

### 3. 직접 실행 (고급)

```bash
npm test           # 전체 테스트 실행 (※ 아직 안 푼 문제는 실패로 표시됩니다 — 정상)
npm run test:watch # 워치 모드 (코드를 고칠 때마다 자동 재실행)

# 특정 문제만 실행 (폴더명 일부로 필터)
npx vitest 다익스트라
npx vitest 8.3-섬의개수
```

## 진행 순서 추천

Lv0 → Lv10 순서대로 풀되, 막히면 `solution.test.ts`의 케이스로 기대 동작을 확인하세요.
코딩이 처음이라면 Lv0(언어 기본기)부터, 문법이 익숙하다면 Lv1부터 시작하세요.
